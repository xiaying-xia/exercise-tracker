# 运动打卡

个人用的运动日程 PWA：每日运动清单、打勾完成、每项可选时长，自动统计**今日**与**本周**运动总时长。数据仅保存在浏览器本地。

## 手机随时打开（任意网络 / 4G）

局域网 `npm run dev` **不能**做到随时访问。需要部署到公网，获得固定 **https://** 地址。

**Vercel 打不开（国内常见）** → 用 **[docs/部署-国内.md](docs/部署-国内.md)**：

- **GitHub Pages**（已配好自动部署，推代码即可）
- 或 `npm run build` 后把 **`dist`** 上传到腾讯云 / 阿里云静态托管

能打开 Vercel 时见 **[docs/手机随时访问.md](docs/手机随时访问.md)**。

> 部署后数据在本机浏览器；换机可在「身体」页底部 **导出/导入备份**。

## 功能

- 添加运动（名称 + 可选分钟数）
- **运动模板（可选）**：内置 70+ 常见项目（有氧/力量/球类等），点一下加入今日；可保存「我的模板」
- 点击圆圈打勾 / 取消
- 点击条目可编辑名称和时长
- 顶部显示：今日时长、本周累计、完成数量
- **身体**：体重、胸围/腰围/臀围（三围），按日记录，可补录历史
- **趋势**：按 日 / 周 / 月 / 自定义周期查看运动总时长与身体数据变化（含简单折线图）
- 左右切换日期，查看历史
- 仅**已打勾且填写了分钟**的项目计入时长

## 本地运行

```bash
npm install
npm run dev
```

终端会打印完整地址，请用 **带 `http://` 的链接** 打开（不要只输入 `localhost`）：

- 本机：**http://localhost:5173/**

### 手机访问（仅开发调试，需同 WiFi）

**不要在手机里输入 `127.0.0.1` 或 `localhost`**——在手机上它们指的是**手机自己**，不是电脑。

1. 电脑终端执行：
   ```bash
   npm run dev:mobile
   ```
2. 终端会打印 **http://192.168.x.x:5173/** 或 **http://11.x.x.x:5173/**（以你电脑局域网 IP 为准）
3. 手机连**与电脑相同的 WiFi**（公司网可能要同一 VPN）
4. 在手机 Safari 里输入终端打印的完整地址

### 电脑本机访问

`npm run dev` → **http://localhost:5173/** 或 **http://127.0.0.1:5173/**（仅电脑有效）

Safari 地址栏可能只显示 `localhost`，点一下地址栏可见完整 `http://...`。

生产预览（build 之后）：

```bash
npm run build
npm run preview
```

访问 **http://localhost:4173/**

## 安装到手机

1. 先 `npm run build` 再 `npm run preview`，或部署到任意 HTTPS 静态托管
2. **iPhone Safari**：分享 →「添加到主屏幕」
3. **Android Chrome**：菜单 →「添加到主屏幕」或「安装应用」

开发模式 (`npm run dev`) 也可添加到主屏幕，但正式使用建议 build 后的版本。

## 配置文件夹（改模板 / 三围不用动代码）

可编辑 **`public/config/`** 下的 JSON，刷新页面即生效。详见 [public/config/README.md](public/config/README.md)。

```
public/config/
  active.json      ← 改 "set" 切换配置套（文件夹名）
  default/         ← 默认
  custom/          ← 示例精简版
```

## 数据存在哪（推荐方式）

| 类型 | 存放位置 | 怎么改 |
|------|----------|--------|
| 运动模板、三围字段、应用标题 | `public/config/<套名>/` | 编辑 JSON，改 `active.json` 切换套 |
| 每日打卡、体重记录 | 浏览器 **localStorage** | 仅本机；换手机需以后做导出（可再加） |

不建议把个人记录也放进 `config` 文件夹——浏览器无法直接读写你电脑上的任意目录；个人数据用 localStorage（或以后的导出 JSON）更合适。

## 技术

- Vue 3 + Vite
- localStorage 持久化
- vite-plugin-pwa（可离线打开）
