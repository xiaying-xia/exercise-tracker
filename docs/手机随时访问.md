# 手机随时打开（不限 WiFi）

`npm run dev` 只在**同一 WiFi** 下能用。若要 **4G / 任意 WiFi 都能开**，需要把应用**部署到公网**，得到一个固定的 **https://** 地址。

本应用是纯前端 + 本地存储，**免费静态托管**即可，无需服务器。

> **Vercel 在国内常打不开？** 请看 **[部署-国内.md](./部署-国内.md)**（GitHub Pages、上传 dist 到腾讯云等）。

---

## 推荐：Vercel（需能打开 vercel.com）

### 1. 构建

```bash
cd ~/exercise-tracker
npm install
npm run build
```

### 2. 部署

[注册 Vercel](https://vercel.com) 后，在终端执行：

```bash
npx vercel
```

按提示登录、选项目目录，首次选默认即可。完成后会给出地址，例如：

`https://exercise-tracker-xxx.vercel.app`

### 3. 手机使用

1. 用手机 Safari / Chrome 打开上面的 **https** 链接（任意网络均可）
2. **添加到主屏幕** → 以后像 App 一样点开
3. 首次打开后，PWA 可**离线**使用（已缓存的页面）

更新代码后重新部署：

```bash
npm run build && npx vercel --prod
```

## 其他免费托管（任选）

| 平台 | 特点 |
|------|------|
| [Cloudflare Pages](https://pages.cloudflare.com) | 国内访问往往更稳定 |
| [Netlify](https://netlify.com) | 与 Vercel 类似 |
| GitHub Pages | 需把项目推到 GitHub |

构建命令均为：`npm run build`，发布目录均为 **`dist`**。

## 重要说明

### 数据存在哪

部署后，打卡、体重等数据仍在**该手机浏览器**的 localStorage 里：

- 不依赖你和电脑是否同一网络
- **换手机** = 新数据（除非以后做账号/云同步）
- 电脑浏览器和手机浏览器的数据**不自动同步**

### 和本地开发的区别

| 方式 | 手机能否随时开 | 数据 |
|------|----------------|------|
| `npm run dev:mobile` + 局域网 IP | 仅同 WiFi | 存在手机浏览器 |
| 部署到 https 地址 | ✅ 任意网络 | 存在手机浏览器 |

### 配置文件夹

`public/config/` 会随构建一起发布；改配置后需重新 `npm run build` 并再部署一次。

## 想要只有你自己能访问？

可在 Vercel 项目设置里开启 **Password Protection**（部分套餐），或使用 Cloudflare Access。个人自用通常直接用长链接即可，不公开分享。
