# 国内部署指南（Vercel 打不开时）

Vercel 网站在国内经常无法访问，**不必强求 Vercel**。下面几种方式都能让手机用 **https** 随时打开。

---

## 方案一：GitHub Pages（推荐，全自动）

适合：能打开 [github.com](https://github.com)（或开代理）的用户。

### 1. 把代码推到 GitHub

在 Mac 终端：

```bash
cd ~/exercise-tracker
git add .
git commit -m "准备部署"
```

在 GitHub 网页 **New repository** 新建仓库，例如叫 `exercise-tracker`（不要勾选多余模板）。

按 GitHub 提示添加远程并推送，例如：

```bash
git remote add origin https://github.com/你的用户名/exercise-tracker.git
git branch -M main
git push -u origin main
```

### 2. 开启 Pages

1. 打开仓库 → **Settings** → **Pages**
2. **Build and deployment** → Source 选 **GitHub Actions**
3. 等几分钟，到 **Actions** 里看绿色勾
4. 回到 **Pages**，会显示地址，例如：  
   `https://你的用户名.github.io/exercise-tracker/`

### 3. 手机使用

用 Safari 打开上面的 **https** 链接 → **添加到主屏幕**。

以后改代码只要 `git push`，会自动重新部署。

---

## 方案二：只上传文件夹（不依赖国外网站）

适合：不想用 GitHub / Vercel，有国内云账号。

### 1. 在本机构建

```bash
cd ~/exercise-tracker
npm run build
```

会生成 **`dist`** 文件夹（里面就是要发布的全部文件）。

### 2. 上传到静态托管

任选一家（控制台里找「静态网站」「对象存储静态托管」）：

| 平台 | 说明 |
|------|------|
| 腾讯云 COS | 开启静态网站，上传 `dist` 内所有文件 |
| 阿里云 OSS | 静态网站托管，上传 `dist` |
| 七牛云 / 又拍云 | 静态空间 + 绑定域名或测试域名 |

上传后会给一个 **https** 访问地址（或自己绑域名）。

### 3. 注意

- 要上传 **`dist` 里面的内容**，不是整个项目文件夹
- 包含 `index.html`、`assets/`、`config/` 等

---

## 方案三：Cloudflare Pages（命令行，少开网页）

若 **vercel.com 打不开** 但终端能连外网，可试：

```bash
cd ~/exercise-tracker
npm run build
npx wrangler pages deploy dist --project-name=exercise-tracker
```

首次会要求在浏览器登录 Cloudflare（若 cloudflare.com 也打不开，用方案一或二）。

---

## 方案四：临时用内网穿透（不推荐长期）

[cpolar](https://www.cpolar.com)、花生壳等，把本机 `5173` 映射成公网 https 地址。

- 优点：不用部署
- 缺点：电脑要开着、免费版地址会变、不稳定

只适合临时演示，**不适合**「随时当 App 用」。

---

## 对比

| 方案 | 手机任意网络 | 要不要电脑开着 | 国内常见难度 |
|------|----------------|----------------|--------------|
| 局域网 dev | ❌ 要同 WiFi | 要 | 低 |
| Vercel | ✅ | 不要 | 网站常打不开 |
| **GitHub Pages** | ✅ | 不要 | 需能访问 GitHub |
| **上传 dist 到国内云** | ✅ | 不要 | 需云账号 |
| 内网穿透 | ✅ | **要** | 中 |

---

## 数据提醒

无论哪种部署，打卡数据都在**手机浏览器本地**。换手机用 **身体** 页里的 **导出/导入备份**。

---

## 仍想试 Vercel？

需能访问 vercel.com（代理 / 海外网络）。能打开后再用：

```bash
npm run build
npx vercel --prod
```

打不开就按本文 **方案一或二** 即可。
